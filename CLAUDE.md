# CLAUDE.md

이 파일은 `H:\내 드라이브\Dev\Aiwebapp_260618`에서 VS Code 기반 에이전트가 먼저 읽어야 하는 작업 지침이다.

## 프로젝트 목표

AI 교육용 웹앱을 만든다. 목표는 단순 AI 도구 체험이 아니라, AI 개념의 작동 원리를 학습자가 조작하고 관찰하게 하는 것이다.

핵심 흐름은 다음이다.

1. 개념 도입
2. 은유적 시뮬레이션
3. 실제 작동 관찰
4. 오류·편향·한계 분석
5. 생활·교과·업무 적용 과제

## 절대 조건

- 인터넷 접속이 없다고 가정한다.
- 외부 사이트 링크는 `07_link_research/verified_links.md`를 기준으로만 사용한다.
- 원천자료는 `00_source_preservation`에 보존되어 있다. 내용을 임의로 생략했다고 판단되면 먼저 이 폴더를 확인한다.
- 기능을 만들 때 설명용 UI와 학습용 질문을 함께 만든다.
- 단순 “AI 체험 링크 모음”으로 만들지 않는다.
- 초등 학습자도 이해할 수 있게 하되, 개념 정확성을 희생하지 않는다.

## 구현 우선순위

1. 작동하는 작은 시뮬레이션
2. 학습자가 조작할 수 있는 파라미터
3. 결과가 변하는 이유 설명
4. 오답·한계·편향 관찰
5. 적용 과제

## 6개 에이전트 역할

- Agent 1: 제품 총괄·요구사항 통합
- Agent 2: 교육과정·콘텐츠 설계
- Agent 3: UX/UI·학습 상호작용 설계
- Agent 4: ML·시뮬레이션 구현
- Agent 5: 전통적 AI·에이전트·피지컬 AI 구현
- Agent 6: QA·문서·통합 관리

각 에이전트의 상세 지시는 `01_agents/agent1.md`부터 `agent6.md`까지를 따른다.

## 작업 방식

1. 작업 전 관련 md를 읽는다.
2. 변경 전 산출물 위치를 명시한다.
3. 한 번에 너무 큰 기능을 만들지 않는다.
4. 각 모듈은 다음 최소 구조를 가진다.
   - 학습 목표
   - 조작 UI
   - 시각화 영역
   - 개념 설명
   - 실험 질문
   - 적용 과제
   - 한계와 오개념 방지
5. 구현 후 acceptance criteria를 갱신한다.

## 인터넷 미접속 대응

외부 사이트를 직접 임베드하지 않는다. 필요한 경우 다음 방식 중 하나를 택한다.

| 상황 | 대응 |
|---|---|
| 외부 시뮬레이션이 핵심 개념을 보여줌 | 자체 축소판 구현 |
| 외부 사이트가 동적 페이지라 크롤링 어려움 | 스크린샷 또는 설명 문서 기반 대체 UI |
| 모델 파일 다운로드 필요 | `public/models`에 사전 배치된 파일만 사용 |
| API 키 필요 | MVP에서는 사용 금지 |
| 외부 교육자료 참조 필요 | `07_link_research/verified_links.md`에 기록된 URL만 문서 링크로 제공 |

## 콘텐츠 톤

- 초등용 설명은 짧고 구체적으로 쓴다.
- 중등 이상 설명은 알고리즘·상태·데이터·평가 개념을 명확히 쓴다.
- 성인용 설명은 업무 적용과 검증 책임을 함께 다룬다.
- “AI가 생각한다” 같은 표현은 피하고, “데이터와 규칙을 바탕으로 예측한다”로 설명한다.

한 모듈이 완료되려면 다음을 만족해야 한다.
