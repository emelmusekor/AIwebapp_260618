# 교육과정 전략

## 전체 학습 경로

```text
AI가 맞히고 틀림
→ 데이터를 모아 AI를 가르침
→ 알고리즘마다 판단 방식이 다름
→ 정답 없이 구조를 찾음
→ 보상으로 행동을 배움
→ 신경망은 가중치와 층으로 패턴을 배움
→ CNN은 이미지를 작은 특징으로 봄
→ RNN/LSTM은 순서를 기억함
→ Transformer/LLM은 토큰 관계와 다음 토큰을 다룸
→ 생성형 AI는 새로운 데이터를 만듦
→ 전통적 AI는 상태·규칙·조건·계획으로 답을 찾음
→ 에이전트와 피지컬 AI는 목표·도구·센서·행동으로 확장됨
```

## 대상별 경로

| 대상 | 핵심 경험 | 권장 모듈 |
|---|---|---|
| 초1–2 | AI가 맞히고 틀리는 관찰 | 낙서 인식, 오답 도감, 음악·그림 AI |
| 초3–4 | 데이터로 AI를 가르침 | AI for Oceans형 분류, Teachable Machine형 활동 |
| 초5–6 | 내가 만든 모델을 게임·기기와 연결 | Scratch/Entry형 블록, micro:bit 움직임 |
| 중등 | 알고리즘 비교와 상태·조건 사고 | 결정경계, K-Means, 미로 탐색, CSP |
| 고등 | 신경망·CNN·LLM·RL 구조 이해 | MLP, CNN, Transformer, GridWorld |
| 대학 | 코드·평가·윤리·캡스톤 | PyTorch/TF.js 연계, RAG, fairness, 프로젝트 |
| 성인 | 업무 적용과 결과 검증 | 문서 요약, 프롬프트, NotebookLM형 RAG, 편향 검증 |

## 모듈 공통 템플릿

각 모듈은 다음 9개 요소를 가진다.

1. 학습 질문
2. 개념 은유
3. 조작 변수
4. 시각화 장면
5. 실험 절차
6. 결과 해석
7. 오답·한계 관찰
8. 실제 적용 예시
9. 평가 또는 산출물

## 학년군별 표현 방식

| 학년군 | 표현 원칙 |
|---|---|
| 초등 저학년 | “AI가 왜 이렇게 생각했을까?”보다 “AI가 왜 이렇게 맞히거나 틀렸을까?”로 접근 |
| 초등 고학년 | 데이터 수집, 라벨, 테스트, 고치기 흐름 강조 |
| 중등 | 상태, 규칙, 알고리즘, 비교 실험 강조 |
| 고등 이상 | 수식·코드·시각화 연결 가능 |
| 성인 | 업무 문제, 위험, 검증 책임 강조 |

## 평가 산출물 예시

| 영역 | 산출물 |
|---|---|
| 지도학습 | 내가 만든 이미지·소리 분류 모델 보고서 |
| 비지도학습 | 우리 반 취향 군집 지도 |
| 강화학습 | 보상 설계 보고서 |
| CNN | AI가 숫자를 헷갈린 이유 분석 |
| LLM | 토큰화와 attention 분석 카드 |
| 전통적 AI | 미로 탐색 알고리즘 비교표 |
| 윤리 | AI 결과 검증 체크리스트 |
