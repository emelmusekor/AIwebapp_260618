# 프로젝트 브리프

## 요청의 핵심

교수님 요청은 “기존 인공지능 시뮬레이션 모방”에서 출발하지만, 단순 복제나 링크 큐레이션이 아니다. 기존 도구의 교육적 구조를 분석해 자체 웹앱 모듈로 재구성하는 것이 핵심이다.

## 제품 콘셉트

AI 개념을 조작형 시뮬레이션으로 이해하는 웹앱.

학습자는 다음 흐름을 반복한다.

1. 문제 상황을 본다.
2. 데이터를 만들거나 상태를 조작한다.
3. AI 또는 알고리즘이 작동하는 과정을 본다.
4. 결과가 맞거나 틀린 이유를 설명한다.
5. 실제 생활·교과·업무 문제로 응용한다.

## 핵심 교육 철학

- AI는 마법이 아니라 데이터, 상태, 규칙, 보상, 계산 구조로 작동한다.
- 학생은 AI를 “써보는 사람”이 아니라 “가르치고, 실험하고, 검증하는 사람”이 되어야 한다.
- 초등 단계에서는 오답 관찰과 데이터 감각이 중요하다.
- 중등 단계에서는 알고리즘별 판단 방식 비교가 중요하다.
- 고등·대학 단계에서는 모델 구조, 평가, 편향, 일반화, 설명가능성까지 연결해야 한다.
- 성인 교육에서는 업무 적용과 검증 책임을 같이 다뤄야 한다.

## 포함해야 할 큰 영역

| 영역 | 핵심 내용 | 대표 모듈 |
|---|---|---|
| AI 감각 형성 | AI가 맞히고 틀리는 경험 | Quick Draw형 낙서 인식, AI for Oceans형 분류 |
| 지도학습 | 라벨, 학습 데이터, 테스트, 결정경계 | Teachable Machine형 분류, 2D 결정경계 |
| 비지도학습 | 군집, 거리, 중심점, 임베딩 | K-Means, Embedding Map |
| 강화학습 | 상태, 행동, 보상, 정책 | GridWorld Q-learning |
| 신경망 | 퍼셉트론, MLP, 손실, 역전파 | XOR, TensorFlow Playground형 실험 |
| CNN | 합성곱, 필터, 특징맵, 풀링, Grad-CAM | 손글씨 숫자 인식, 필터 실험 |
| RNN/LSTM | 순서, hidden state, gate | 문장 기억 릴레이 시뮬레이션 |
| Transformer/LLM | 토큰, 임베딩, attention, next token | Tokenizer, attention map |
| 생성형 AI | GAN, VAE, Diffusion, 프롬프트 | 노이즈 제거, 이미지 생성 원리 |
| 전통적 AI | 탐색, CSP, 논리, 규칙, 계획 | 미로 탐색, 스도쿠, 전문가 시스템 |
| 에이전틱·피지컬 AI | 도구 사용, 계획, 센서, 행동 | 도구 사용 agent, micro:bit형 움직임 인식 |
| 윤리·평가 | 편향, 환각, 책임, 설명가능성 | 편향 시뮬레이션, 혼동행렬 |

## 산출물 정의

- 웹앱: 학습 모듈 중심 SPA.
- 기획 문서: 모듈별 학습 목표, UI, 조작 변수, 평가 기준.
- 교사용 자료: 질문, 활동, 토론 주제.
- 개발 문서: 데이터 구조, 시뮬레이션 구조, 테스트 기준.

## 핵심 제약

- 모든 외부 링크를 실시간 호출할 수 있다고 가정하면 안 된다.
- 브라우저에서 돌아가는 자체 구현을 우선한다.
- 무거운 딥러닝 모델은 사전 학습 모델, 샘플 데이터, 시각화형 구현으로 대체한다.
- 초등 수업에서 개인정보·카메라·음성 데이터 수집은 보수적으로 다뤄야 한다.
