# 상호작용 모델

## 공통 화면 구조

```text
[모듈 제목]
[핵심 질문]

좌측: 조작 패널
중앙: 시뮬레이션 시각화
우측: 결과 해석 / 개념 설명 / 실험 질문
하단: 적용 과제 / 오답 사례 / 다음 모듈
```

## 조작 유형

| 유형 | 예시 모듈 | 핵심 UI |
|---|---|---|
| 그리기 | 숫자 인식 | Canvas, 지우기, 예측하기 |
| 점 찍기 | 결정경계, K-Means | 좌표평면, 라벨 선택, 알고리즘 선택 |
| 격자 조작 | 미로, GridWorld | 벽, 시작점, 목표점, 보상 |
| 텍스트 입력 | Tokenizer, Attention | 입력창, 토큰 카드, 연결선 |
| 규칙 추가 | 전문가 시스템 | IF 조건 카드, THEN 결론 카드 |
| 슬라이더 | 학습률, temperature | 즉시 반영되는 파라미터 |

## 피드백 방식

- 색상만으로 의미를 전달하지 않는다.
- 숫자, 레이블, 설명 문구를 함께 제공한다.
- “왜 결과가 바뀌었는가?” 문장을 자동으로 표시한다.
- 오답을 실패가 아니라 학습 자료로 보여준다.
