# 강화학습 기획

## 핵심 질문

정답을 알려주지 않아도 에이전트는 보상을 통해 더 나은 행동을 배울 수 있는가?

## 포함 모듈

- GridWorld Q-learning
- 보상 설계 실험
- 탐험과 활용 균형
- 정책 화살표 시각화

## 학습 목표

- 상태, 행동, 보상, 정책의 의미를 설명한다.
- 좋은 보상 설계가 행동을 바꾼다는 것을 관찰한다.
- Q-table과 value heatmap의 의미를 이해한다.
- 단기 보상이 항상 장기 최적 행동을 만들지 않음을 확인한다.

## 주요 상호작용

| 조작 | 결과 |
|---|---|
| 보상 위치 변경 | 정책 변화 |
| 벽 추가 | 탐색 경로 변화 |
| 학습률 변경 | 수렴 속도 변화 |
| 탐험률 변경 | 시행착오 패턴 변화 |
| 에피소드 반복 | Q값 업데이트 관찰 |

## 실제 적용

- 미로 탈출 로봇
- 게임 캐릭터 행동
- 배송 경로 최적화
- 운동 루틴 보상 설계
- 로봇 행동 학습 입문

## 오개념 방지

- 강화학습은 무조건 스스로 똑똑해지는 방식이 아니다.
- 보상이 잘못 설계되면 원하지 않는 행동을 학습한다.
- 충분한 시행착오와 안전한 환경이 필요하다.
