# LLM과 생성형 AI 기획

## 핵심 질문

LLM은 문장을 어떻게 쪼개고, 문맥을 보고, 다음 토큰을 예측하는가?

## 포함 모듈

- 토큰화 실험
- Attention 히트맵
- Next-token prediction 놀이
- 프롬프트와 출력 변화
- RAG와 출처 기반 답변
- Diffusion 노이즈 제거

## 학습 목표

- 토큰은 사람이 느끼는 글자·단어와 다를 수 있음을 이해한다.
- Attention은 문장 안 토큰 간 관계를 계산하는 구조임을 설명한다.
- LLM은 정답 저장소가 아니라 다음 토큰 예측 모델임을 이해한다.
- 생성형 AI 결과는 검증이 필요함을 안다.

## 주요 상호작용

| 조작 | 결과 |
|---|---|
| 문장 입력 | 토큰 분할 표시 |
| 단어 클릭 | 관련 attention 연결선 표시 |
| temperature 변경 | 출력 다양성 변화 |
| top-k/top-p 변경 | 후보 토큰 변화 |
| 근거 문서 추가 | RAG형 답변 변화 |

## 실제 적용

- 문서 요약
- 회의록 정리
- 챗봇 기획
- 자료 검색 보조
- 교육용 피드백 생성

## 오개념 방지

- LLM은 항상 사실을 말하지 않는다.
- 그럴듯한 문장은 검증된 문장과 다르다.
- 프롬프트 능력보다 문제 분해와 검증 능력이 더 중요하다.
