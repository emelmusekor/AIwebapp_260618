# 에이전틱 AI와 피지컬 AI 기획

## 핵심 질문

AI가 단순 답변을 넘어 목표를 세우고, 도구를 쓰고, 실제 세계의 센서와 행동으로 연결되려면 무엇이 필요한가?

## 포함 모듈

- 도구 사용 Agent workflow
- Planner-Executor-Evaluator 구조
- Multi-agent 협업 모형
- micro:bit 움직임 인식
- Edge Impulse형 센서 모델
- 로봇 foundation model 개념 소개

## 학습 목표

- 에이전트는 목표, 도구, 메모리, 계획, 평가 루프를 가진 시스템임을 이해한다.
- 피지컬 AI는 시각·언어 이해만으로 충분하지 않고 센서, 행동, 안전 제약이 필요함을 안다.
- 실제 로봇 제어는 시뮬레이션과 현실 간 차이가 있음을 이해한다.

## 주요 상호작용

| 조작 | 결과 |
|---|---|
| 목표 입력 | 계획 단계 생성 |
| 도구 선택 | 가능한 행동 경로 변화 |
| 실패 조건 추가 | 재계획 발생 |
| 센서 데이터 카드 분류 | 움직임 인식 결과 변화 |
| 안전 규칙 추가 | 행동 제한 |

## 실제 적용

- 수업 보조 챗봇
- 자료 검색 agent
- 청소 로봇 행동 계획
- micro:bit 운동 타이머
- 센서 기반 안전 알림

## 오개념 방지

- 에이전트는 자율성을 가진 것처럼 보여도 설계된 도구와 규칙 안에서 작동한다.
- 피지컬 AI는 현실 안전 문제가 있으므로 교육용 모형과 실제 제어를 구분해야 한다.
- 다중 에이전트는 무조건 성능을 높이지 않으며 조정 비용이 생긴다.
